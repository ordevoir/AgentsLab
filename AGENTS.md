# AgentsLab – Codex Rules

## Общая цель
Ты работаешь как инженер-ассистент над проектом agentslab.
Приоритет: корректность, читаемость, минимальные изменения.

## Git и безопасность
- Проект под git — используй это.
- ВСЕ изменения показывай в виде диффа.
- Никогда не делай `git commit` без явного запроса.
- Не трогай историю git и .git/.

## Рабочий процесс
1. Сначала предложи план.
2. Потом вноси изменения.
3. В конце: краткий список изменённых файлов и зачем.

## Стиль работы
- Не делай массовых рефакторингов без запроса.
- Если есть сомнения — спрашивай.
- Комментарии в коде добавляй только если это повышает понятность.

## Ограничения
- Не добавляй новые зависимости без подтверждения.
- Не меняй конфигурацию CI/CD без явного запроса.
